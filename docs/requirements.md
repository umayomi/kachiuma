# 要件定義

## 1. 目的

netkeiba のデータを使い、各馬の推定勝率と実オッズを比較して
**期待値（EV）の高い馬券候補を可視化する個人用ツール**を作る。
最終的な馬券購入はユーザーが手動で行う。

## 2. 絶対条件（制約）

| # | 制約 | 理由 |
|---|---|---|
| C1 | 馬券購入・投票の自動化は実装しない | 購入は手動。対象外と明言済み |
| C2 | 個人利用のみ。商用化・収益化しない | ToS・依存症配慮 |
| C3 | 原則無料で運用 | GitHub/Actions/Vercel 無料枠＋netkeiba収集 |
| C4 | 開発も使用もスマホで完結できる | 母艦PCを前提にしない |
| C5 | 生データは公開リポジトリにコミットしない | netkeiba ToS のグレーゾーン回避 |
| C6 | 分析の根拠が「わかる」設計 | 数字の羅列にしない |

## 3. 機能要件

### 必須（MVP）
- レース単位で出走馬一覧を表示
- 各馬の **推定勝率**・**市場想定勝率（オッズ由来）**・**期待値**・**印** を表示
- 「**なぜこの印か**」の根拠を1〜2行で提示
- レース選択（日付・競馬場・R）

### あると良い
- 券種別の買い目候補（単勝・複勝・ワイド等。EVベース）
- バックテスト結果（回収率・的中率）の表示
- オッズ変動の反映（締切前の最新オッズで再計算）
- 予想の信頼度（確率キャリブレーションの良し悪し）

## 4. 非機能要件

- スマホ最優先（縦持ち・片手操作）。PCでも崩れない
- 表示は3秒以内目標（静的JSONを読むだけにする）
- アクセシビリティ：キーボードフォーカス可視、`prefers-reduced-motion` 尊重

## 5. 「期待値」の定義（共通言語）

- **推定勝率 p**：分析が出す「その馬が来る確率」
- **市場想定勝率 q**：オッズから逆算した確率（控除前）。`q ≒ 1 / odds`
- **期待値 EV**：単勝なら `EV = p × odds`。`EV > 1.0` で理論上プラス
- **妙味（edge）**：`p − q`。市場より自分が高く見ている度合い

> 注意：控除率（約20〜30%）があるため、`EV > 1.0` を継続して出すのは難しい。
> 過剰な確信を表示しない。あくまで判断材料として提示する。

## 6. スコープ外（やらないこと）

- 馬券の自動購入、IPAT/即PAT連携、入出金の自動化
- 有料データ（JRA-VAN等）を前提とした実装（将来検討の余地は残す）
- 他人への公開・配布を目的とした機能
