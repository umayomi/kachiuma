## 既知の注意点（AIへの申し送り）
- 控除率があるため EV>1 を安定して出すのは難しい。過剰な確信表示をしない。
- netkeiba の HTML 構造は変わることがある。セレクタは壊れやすい前提で、失敗を検知してログに残す。
- Actions の分節約のため、Selenium は requests で取れない時だけ使う。
- 生データは絶対にコミットしない（公開リポジトリ）。
